# Эталон быстродействия

Полный локальный набор benchmark-тестов запускается командой:

```bash
task benchmark
```

Набор использует MCP-транспорты в памяти, вымышленные credentials и локальный тестовый HTTP-сервер. Обращений к Beget нет. Каждый benchmark выводит задержку, объём выделенной памяти и количество allocations. Benchmark-тесты MCP-каталогов также показывают сериализованный размер ответа, а `BenchmarkMCPToolsList` выводит число инструментов.

## Эталонный запуск

Этот диагностический baseline получен 2026-07-13 с Go 1.26.5 на Linux amd64 и AMD Ryzen 9 9950X3D. Значения не являются порогами CI, поскольку планировщик и различия оборудования создают нормальные колебания.

| Benchmark | Время операции | Байт на операцию | Allocations |
| --- | ---: | ---: | ---: |
| Запуск сервера | 12,93 мс | 7 096 877 | 192 975 |
| Инициализация MCP | 123,6 мкс | 308 325 | 215 |
| MCP `tools/list` | 9,510 мс | 6 621 932 | 40 769 |
| MCP capability resource | 205,7 мкс | 228 522 | 108 |
| Самая большая входная схема Cron | 6,541 мкс | 13 374 | 44 |
| Вложенная схема директив | 6,013 мкс | 13 756 | 55 |
| Вызов MCP-инструмента | 106,5 мкс | 340 383 | 256 |
| Конкурентные вызовы MCP-инструмента | 63,14 мкс | 340 472 | 258 |
| Локальный HTTP round trip | 27,37 мкс | 10 302 | 116 |
| Конкурентные локальные HTTP round trip | 32,65 мкс | 36 057 | 146 |

Ответ `tools/list` сохранил все 66 инструментов и занял 144 736 сериализованных байт с отдельными входными и выходными схемами операций. Опциональный capability resource занял 4 729 сериализованных байт, примерно в 31 раз меньше, и читается только при неясном выборе маршрута. Таблица нужна для сравнения результатов профилирования и поиска регрессий, а не как обещание одинаковых значений на другом компьютере.
