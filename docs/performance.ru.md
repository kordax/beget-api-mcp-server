# Эталон быстродействия

Полный локальный набор benchmark-тестов запускается командой:

```bash
task benchmark
```

Набор использует MCP-транспорты в памяти, вымышленные credentials и локальный тестовый HTTP-сервер. Обращений к Beget нет. Каждый benchmark выводит задержку, объём выделенной памяти и количество allocations. `BenchmarkMCPToolsList` дополнительно показывает сериализованный размер ответа и число инструментов.

## Эталонный запуск

Этот диагностический baseline получен 2026-07-13 с Go 1.26.5 на Linux amd64 и AMD Ryzen 9 9950X3D. Значения не являются порогами CI, поскольку планировщик и различия оборудования создают нормальные колебания.

| Benchmark | Время операции | Байт на операцию | Allocations |
| --- | ---: | ---: | ---: |
| Запуск сервера | 3,935 мс | 2 636 676 | 70 932 |
| Инициализация MCP | 138,5 мкс | 309 832 | 217 |
| MCP `tools/list` | 2,767 мс | 1 971 242 | 13 317 |
| Самая большая входная схема Cron | 5,968 мкс | 13 341 | 44 |
| Вложенная схема директив | 5,838 мкс | 13 734 | 55 |
| Вызов MCP-инструмента | 103,4 мкс | 337 658 | 196 |
| Конкурентные вызовы MCP-инструмента | 70,02 мкс | 337 508 | 199 |
| Локальный HTTP round trip | 27,25 мкс | 10 171 | 115 |
| Конкурентные локальные HTTP round trip | 33,83 мкс | 37 779 | 146 |

Ответ `tools/list` сохранил все 66 инструментов и занял 56 615 сериализованных байт. Таблица нужна для сравнения результатов профилирования и поиска регрессий, а не как обещание одинаковых значений на другом компьютере.
